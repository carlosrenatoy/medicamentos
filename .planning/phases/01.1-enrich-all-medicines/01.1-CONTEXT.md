# Phase 1.1: Enrich All Medicines to Gold Standard - Context

**Gathered:** 2026-05-05
**Status:** Ready for planning

<domain>
## Phase Boundary

Upgrade ALL 160 medicines in `src/data.ts` to match the quality level proven with Cefalexina (Keflex) and Adrenalina (Drenalin). This means every medicine must have:

1. **Nome**: `"Princípio Ativo (Nome Comercial)"` — ex: `"Cefalexina (Keflex)"`
2. **Apresentações** (`presentations`): extraídas de `Apresentacao_preenchida` da planilha
3. **Cálculos corretos**: `mgPerKg`, `maxDose`, `divideBy`, `intervalText`
4. **Sem duplicatas, sem quebrar o app, sem alterar layout**
</domain>

<decisions>
## Implementation Decisions (LOCKED)

### Regra #1: O Padrão Ouro
Toda medicação DEVE seguir o modelo da Cefalexina (oral) ou Adrenalina (injetável complexa):

**Medicação Oral (modelo Cefalexina):**
- `presentations` no nível global do Medicine (suspensão, comprimido)
- `divideBy` = número de vezes por dia (4 para 6/6h, 3 para 8/8h, 2 para 12/12h)
- `intervalText` = "de 6/6h", "de 8/8h", "de 12/12h"
- `isPill: true` para comprimidos/cápsulas

**Medicação Injetável Complexa (modelo Adrenalina):**
- `presentations` no nível da DOSE quando a diluição muda por via
- `hideVolumeCalc: true` para infusão contínua e nebulização
- `defaultDrugMg`/`defaultVolume`/`ampouleConcentration_mg_ml` para bomba de infusão

**Medicação Injetável Simples:**
- `presentations` no nível global com a concentração da ampola

### Regra #2: Fonte de Verdade
- Campo `Apresentacao_preenchida` do JSON = descrição das apresentações
- Campo `Concentracoes_para_app` do JSON = concentrações numéricas (quando preenchido)
- Campo `Dose_original` = instruções clínicas
- Campo `Comentário_original` = comentários/alertas
- NÃO inventar apresentações ou concentrações que não estejam na planilha

### Regra #3: Nomes Comerciais
- Mapear pela coluna `Medicamento` do JSON
- Consultar nomes comerciais brasileiros comuns:
  - Amoxicilina → (Amoxil)
  - Ampicilina → (Binotal)
  - Ceftriaxona → (Rocefin)
  - etc.
- Os que já têm nome comercial no data.ts atual: MANTER

### Regra #4: Segurança
- NUNCA usar PowerShell `Set-Content` no data.ts (causa double-encoding)
- Usar somente Node.js `fs.writeFileSync` com encoding 'utf8'
- Rodar `tsc --noEmit` APÓS cada batch de alterações
- Verificar visualmente o app no browser após cada batch
- Bump cache version (v13 → v14+) no App.tsx após alterações

### Regra #5: Não Apagar
- NUNCA excluir uma medicação existente
- NUNCA remover um campo que já funciona
- Apenas ADICIONAR ou ENRIQUECER campos
</decisions>
