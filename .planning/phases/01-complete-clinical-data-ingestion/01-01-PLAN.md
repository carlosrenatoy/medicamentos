# Plan 01-01: Ingest Lote JSONs into data.ts

## Objective
Parse the raw JSON objects from `lote1.json` through `lote8.json`, map them correctly to the new `MedicineDose` structure (including presentations), and append them to `INITIAL_MEDICINES` in `src/data.ts`.

## Context
The previous AI instance chunked the source Excel data into 8 JSON files to avoid context exhaustion, but never merged them into the actual application catalog.

## Verification
- `npm run lint` passes without type errors.
- The `INITIAL_MEDICINES` array contains approximately 140 elements.
- The UI search renders all 140 items without throwing exceptions.

## Must Haves
- Data must strictly map to `Medicine`, `MedicineDose`, and `DosePresentation` interfaces.
- No legacy structures from previous phases should be injected.
- Duplicate names must be handled or skipped.

## Tasks

<task>
<read_first>
- src/types.ts
- src/data.ts
- lote1.json
</read_first>

<action>
Create a standalone Node/TypeScript script at `scripts/ingest_lotes.ts` that:
1. Reads all files matching `lote*.json` in the root directory.
2. Iterates over the arrays of raw objects inside them.
3. For each object, constructs a `Medicine` object compatible with `src/types.ts`.
4. Parses the `Concentracoes_para_app` field and creates the necessary `DosePresentation` objects.
5. Emits a new combined JSON file `final_medicines.json` or modifies `src/data.ts` directly.
Note: Since `data.ts` is a TypeScript file with an exported array, the script should use string manipulation or AST parsing, or alternatively, the execution agent can run the script to get the JSON output and then insert the JSON array into `src/data.ts` manually using `write_to_file`/`replace_file_content`.
</action>

<acceptance_criteria>
- `scripts/ingest_lotes.ts` exists.
- The script correctly parses the JSON schema seen in `lote1.json`.
</acceptance_criteria>
</task>
